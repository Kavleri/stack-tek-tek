# Tugas Proyek Akhir untuk mata kuliah Fullstack Web Development:

### 1. Bobot dan Pentingnya Proyek
Tugas Proyek ini merupakan komponen penilaian terbesar, yaitu **50% dari total nilai akhir**. Syarat kelulusan minimal untuk mata kuliah ini adalah nilai 60 (C), sehingga keberhasilan proyek ini sangat krusial.

### 2. Ketentuan Pembentukan Tim
*   **Komposisi:** Setiap kelompok terdiri dari **5 orang mahasiswa**.
*   **Sifat Tim:** Tim bersifat tetap dan tidak dapat diganti hingga akhir semester (Pertemuan 1-14).
*   **Peran:** Seluruh anggota wajib berperan sebagai **Fullstack Developer**, artinya setiap mahasiswa harus mengerjakan aspek frontend dan backend secara kolaboratif.
*   **Administrasi:** Nama anggota tim harus dikirimkan ke platform **elena** dalam format `.txt`. Setiap perkembangan akan didampingi oleh asisten dosen.

### 3. Persyaratan Teknis Proyek
Proyek harus dibangun dari awal (end-to-end) dengan ketentuan sebagai berikut:
*   **Backend:** Wajib membangun sistem **RESTful API** menggunakan **Node.js dan Express.js** dengan arsitektur terstruktur (**MVC**) dan terintegrasi dengan database.
*   **Frontend:** Membangun antarmuka aplikasi berbasis **React.js** yang terintegrasi dengan backend API yang telah dibuat.
*   **Fitur:** Setiap anggota kelompok minimal harus membangun **satu fitur** yang mencakup sisi backend dan frontend sekaligus.

### 4. Alur Kerja dan Tugas (Sprint)
Proyek ini dikerjakan dengan metodologi **Agile/Scrum**. Berdasarkan panduan Sprint Minggu ke-2:
*   Kelompok harus menentukan **judul proyek**.
*   Menentukan **daftar fitur** yang akan dikembangkan.
*   Memasukkan pembagian tugas atau **backlog** ke dalam template sprint yang sudah disediakan.
*   Menyusun laporan sesuai format di elena dan mengumpulkannya kepada asisten dosen serta melalui sistem elena.

### 5. Standar Kualitas yang Diharapkan
Dosen menekankan beberapa prinsip penting dalam pengerjaan proyek, antara lain:
*   **Keamanan:** Menerapkan prinsip keamanan (OWASP) sejak awal, seperti penggunaan JWT untuk autentikasi dan pencegahan SQL Injection.
*   **Kualitas Kode:** Menulis kode yang dapat dipelihara (*maintainable*), aman, dan teruji (*testable*).
*   **Deployment:** Proyek diharapkan dapat dideploy ke publik (misalnya menggunakan Vercel, AWS, atau VPS) dan dilengkapi dengan *pipeline* CI/CD seperti GitHub Actions.
